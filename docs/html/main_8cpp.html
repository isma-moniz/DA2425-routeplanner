void calcBestDrivingRoute ( )

Calculates the best driving route between two locations using Dijkstra's algorithm.

This function asks the user for a source and destination location ID, then calls Dijkstra's algorithm via the storageHandler to calculate the best driving route.

Time complexity: O(E log V), where E is the number of edges and V is the number of vertices. This is due to the Dijkstra's algorithm used in storageHandler.callDijkstra().

◆ calcDrivingRouteRestricted()

void calcDrivingRouteRestricted ( )

Calculates the best driving route with restrictions.

This function allows the user to input source and destination IDs, as well as avoid nodes and segments, and then calls the restricted Dijkstra's algorithm to calculate the route with the given restrictions.

Parameters
srcThe source location ID.
destThe destination location ID.
avoidNodesA string of node IDs to avoid, separated by commas.
avoidSegmentsA string of segment IDs to avoid, separated by commas.
includeNodeA string representing a node to include in the route.

Time complexity: O(E log V), where E is the number of edges and V is the number of vertices. The complexity is similar to that of Dijkstra's algorithm but with additional constraints.
